# Explanation of Commands

## Command 1

**Command**

```bash
touch system.log
```

**Explanation**

The `touch` command creates an empty log file named `system.log`. This file will be continuously monitored for new log entries.

---

## Command 2

**Command**

```bash
cat > system.log << EOF
...
EOF
```

**Explanation**

This command writes sample log entries into the log file. The entries include both normal information messages and error messages for testing.

---

## Command 3

**Command**

```bash
tail -f system.log | grep "ERROR" >> error_report.txt 2> /dev/null
```

**Explanation**

This command continuously monitors the log file using `tail -f`. Newly added lines are passed through the pipe to `grep`, which filters only `ERROR` messages and appends them to `error_report.txt`. Any error messages generated by the command itself are redirected to `/dev/null`, preventing unnecessary output.

---

## Command 4

**Command**

```bash
echo "INFO: User logged out" >> system.log
```

**Explanation**

This command appends a new informational log entry to the log file. Since it does not contain the word `ERROR`, it is ignored by the monitoring pipeline.

---

## Command 5

**Command**

```bash
echo "ERROR: Network Timeout" >> system.log
```

**Explanation**

This command appends an error message to the log file. The monitoring pipeline immediately detects and stores it in `error_report.txt`.

---

## Command 6

**Command**

```bash
echo "ERROR: Permission Denied" >> system.log
```

**Explanation**

This command adds another error message to the log file. The pipeline extracts it automatically and appends it to the report file.

---

## Command 7

**Command**

```bash
cat error_report.txt
```

**Explanation**

The `cat` command displays all extracted error messages stored in the report file, verifying that the monitoring pipeline worked correctly.

---

## Command 8

**Command**

```bash
cat error_report.txt > output.txt
```

**Explanation**

This command copies the contents of the report file into `output.txt`. It preserves the final output for submission.
